# Why These Skills? (Quick Reference)
| Skill | Why It Matters for This Project |
|---|---|
| **frontend-design** | Prevents the #1 risk: generic, forgettable UI. Enforces distinctive typography, spatial composition, and motion. |
| **taste-skill** | The user's core requirement is "great taste." This skill tunes visual density, motion intensity, and design variance — the difference between "competent" and "exquisite." |
| **Superpowers** | A multi-page Astro project with CMS, 3 content types, cross-linking, and auth is complex. Structured workflow prevents chaos. |
| **Vercel Web Design Guidelines** | Spacing scales, type hierarchies, responsive breakpoints — the invisible infrastructure of a polished site. |
| **UI/UX Pro Max** | Font pairing exploration. With a warm brown editorial palette, the right serif + sans-serif pairing is everything. |
| **UI Skills pipeline** | The 4-step polish pass (baseline → a11y → motion → performance) is the closest thing to a "QA department in a box." |
| **wondelai/skills** | Content platforms live or die on information architecture. Norman's principles ensure the FAQ → Video → Syllabus navigation feels intuitive, not overwhelming. |
| **Creative Director** | Elevates the project from "developer-built website" to "designed experience." Cannes-calibrated methodology. |
| **AccessLint** | A public educational platform for Indian citizens must meet accessibility standards. Warm brown on cream requires careful contrast validation. |
| **Beautiful Prose** | Every label, button, tooltip, and empty state message should read like it was written by an editor, not generated by AI. |
# Workflow Order
1. **Superpowers** → Plan the full architecture (pages, components, content schema, CMS)
2. **Creative Director** → Establish the conceptual design direction
3. **frontend-design** + **taste-skill** → Build with distinctive, tasteful aesthetics
4. **UI/UX Pro Max** → Validate font pairings and palette harmony
5. **wondelai/skills** → Audit information architecture and user flows
6. **UI Skills pipeline** → Polish pass (baseline → accessibility → motion)
7. **Design Auditor** + **AccessLint** → Final quality audit
8. **seo-audit** → SEO pass before launch
